# タンク！Web版 要件定義書

> 『はじめてのWii（Wii Play）』のミニゲーム「タンク！」をWebアプリで再現するための要件定義書。
> 本書はClaude Codeでの実装を前提に、仕様を実装可能な粒度で記述する。

---

## 1. プロジェクト概要

| 項目 | 内容 |
|---|---|
| プロジェクト名 | タンク！Web版（Wii Tanks Clone） |
| 目的 | 名作ミニゲーム「タンク！」をブラウザ上でフル再現する |
| 成果物 | 単一HTMLファイル（HTML+CSS+JavaScript同梱）で動作するWebゲーム |
| 対象ブラウザ | 最新版 Chrome / Edge / Firefox / Safari（デスクトップ） |
| 想定プレイヤー | PCキーボード＋マウスでプレイするユーザー |

### 1.1 開発方針

- **単一ファイル構成**: `index.html` 1ファイルに HTML / CSS / JavaScript をすべて含める。外部ライブラリ・ビルドツール不要。ブラウザで直接開けば動く。
- **描画**: HTML5 `<canvas>` + Vanilla JS（2Dコンテキスト）。フレームワーク・ゲームエンジンは使わない。
- **物理・当たり判定**: 自前実装（AABB矩形判定＋反射ベクトル計算）。
- **コードは可読性重視**: 機能ごとに関数・クラスを分割し、コメントを付す（後述のモジュール構成参照）。

---

## 2. スコープ

**フル再現**を目標とする。原作の以下の要素をすべて実装する。

- 全9種類の敵戦車（茶・灰・青緑・黄・ピンク・緑・紫・白・黒）と、それぞれ固有のAI挙動
- 砲弾の跳弾（リコシェ）、地雷、破壊可能ブロック
- 1人プレイ（全100ミッション相当、残機制、5ミッションごとに残機+1）
- ローカル2人対戦（ミッション20まで、より多く倒した方が勝ち）
- ミッション21以降のランダム生成、10の倍数の固定盤面

### 2.1 対応プレイモード

| モード | 内容 | ステージ数 |
|---|---|---|
| 1人プレイ（対COM） | 原作メイン。残機制でミッションを連続クリア | 100 |
| ローカル2人対戦 | 同一キーボードで2人同時プレイ。COM撃破数を競う | 20 |

> オンライン対戦は対象外。

## 3. 操作仕様（キーボード＋マウス）

Wiiリモコンが使えないため代替操作とする。

### 3.1 1人プレイ（P1）

| 操作 | 入力 |
|---|---|
| 移動（上下左右） | `W` `A` `S` `D` |
| 照準 | マウスカーソル位置（自機からカーソルへ砲塔が向く） |
| 砲弾発射 | マウス左クリック |
| 地雷設置 | マウス右クリック（または `Space`） |
| ポーズ | `Esc` または `P` |

### 3.2 ローカル2人対戦

| 操作 | P1（青） | P2（赤） |
|---|---|---|
| 移動 | `W` `A` `S` `D` | 方向キー `↑` `↓` `←` `→` |
| 砲塔回転 | マウス | `,`（左回転）/ `.`（右回転） |
| 砲弾発射 | マウス左クリック | `/` または `Enter` |
| 地雷設置 | `Space` | `Shift`（右） |

> 2人対戦時はP2が砲塔角度をキーで回す方式とする。実装簡略化のため「自機の進行方向に砲塔を向ける」自動エイムをオプションで用意してもよい（設定で切替）。

## 4. ゲームルール仕様

### 4.1 共通ルール

- フィールド上のすべての敵戦車を破壊するとミッションクリア。
- 自機・敵を問わず、**砲弾または地雷が1発当たると即破壊**。
- フィールドは長方形のアリーナ。内部にブロック（壁）と、戦車は通れないが砲弾は通過する「穴／溝」を配置できる。

### 4.2 1人プレイ

- 開始時の残機は **3**。
- 被弾で1ミス、残機0でゲームオーバー。
- **5ミッションごとに残機+1**。
- 最初はミッション1〜20のみ。20クリアで全100ミッションが解放。
- ミッション50で黒戦車が登場。
- **ミッション21以降、10の倍数以外はランダム生成**（過去の盤面を流用し、敵配置を変える）。10の倍数は固定の作り込み盤面。

### 4.3 2人対戦

- ミッション20まで。
- 各プレイヤーの残機は各ミッション1。両者が同じミッションで倒れるとゲームオーバー。
- 片方がクリアすれば、もう片方が倒れていても次へ進む。
- COM（敵戦車）をより多く破壊した方が勝ち。

### 4.4 弾・地雷の仕様

| 項目 | 自機の仕様 |
|---|---|
| 同時発射弾数 | 画面上に最大5発。上限到達中は弾が消えるまで撃てない |
| 跳弾回数 | 壁で1回跳ね返る。2回目の壁接触で消滅 |
| 地雷 | 同時に最大2個設置可能 |
| 地雷の爆発 | 設置から一定時間（約10秒）で自動爆発。戦車が近づく／砲弾が当たると即爆発 |
| 破壊可能ブロック | 地雷の爆発・特定の砲弾で破壊可能（ピンク／破壊可ブロック） |

- 砲弾同士は衝突して相殺できる。
- 砲弾で離れた地雷を誘爆できる。
- 砲弾が穴／溝の上を通過できる（戦車は通れない）。

## 5. 敵戦車仕様（AI）

各敵は以下の性能パラメータとAI挙動を持つ。**この表が実装の中核**。

### 5.1 性能パラメータ表

| 戦車 | 初登場 | 移動速度 | 同時弾数 | 跳弾回数 | 地雷数 | 知能 | 弾種 |
|---|---|---|---|---|---|---|---|
| 自機(P1/P2) | Lv.1 | 普通 | 5 | 1 | 2 | プレイヤー | 砲弾＋地雷 |
| 茶 Brown | Lv.1 | 不動 | 1 | 1 | 0 | 低 | 砲弾 |
| 灰 Grey | Lv.2 | 低速 | 1 | 1 | 0 | 低 | 砲弾 |
| 青緑 Teal | Lv.5 | 低速 | 1 | 0 | 0 | 中〜高 | ロケット |
| 黄 Yellow | Lv.8 | 高速 | 1 | 1 | 4 | 極低 | 砲弾＋地雷 |
| ピンク Pink | Lv.10 | 普通 | 3 | 1 | 0 | 中 | 砲弾 |
| 緑 Green | Lv.12 | 不動 | 2 | 2 | 0 | 極高 | 跳ねるロケット |
| 紫 Purple | Lv.15 | 高速 | 5 | 1 | 2 | 高 | 砲弾＋地雷 |
| 白 White | Lv.20 | 普通 | 5 | 1 | 2 | 中〜低 | 砲弾＋地雷 |
| 黒 Black | Lv.50 | 極高速 | 2 | 0 | 2 | 極高 | ロケット＋地雷 |

> 移動速度の目安: 不動=0 / 低速=1.0 / 普通=1.6 / 高速=2.2 / 極高速=3.0（px/frame, 60fps基準。実装時に調整）。

### 5.2 各敵のAI挙動

**茶 Brown** — 不動。砲塔はランダムに回転し、射線に自機が入ると1発撃つ。地雷なし。最弱。

**灰 Grey** — 低速で適当に移動（壁に当たると向き変更）。砲塔は緩く自機を追う。1発ずつ撃つ。

**青緑 Teal** — 低速・防御的移動（弾や地雷を避ける）。砲塔は強く自機を狙い、ロケットを1発（跳弾なし）。弾速が速い。自機が動き続けると外しやすい＝「現在位置」を狙う実装にする。

**黄 Yellow（自爆兵）** — 高速・適当移動。地雷を最大4個、頻繁に無秩序に設置。地雷を強く避ける（自分の地雷で自滅し得る）。時々自機の射線に突っ込む“自爆”挙動。

**ピンク Pink** — 普通速度で**攻撃的に自機へ接近**。砲塔・移動とも自機を追う。3発まで撃つ。地雷なし。

**緑 Green** — 不動。砲塔は強く自機を狙い、**壁の跳弾を高精度で計算**して2回跳ねるロケットを2発撃つ。直接撃つことはほぼなく、跳弾で狙う。

**紫 Purple** — 高速。射線が通る時のみ攻撃的。「跳弾で牽制→接近して攻撃」のリコシェ＆アタック戦法。紫同士で連携（別角度から挟む）。無駄に地雷を置いて自滅しがち。5発・地雷2個。

**白 White** — **ミッション開始直後に透明化**。移動時に**履帯（タイヤ跡）を残し位置が分かる**。透明化時に高音=攻撃的/低音=防御的のSE。5発・地雷2個。挙動は紫に近いがやや知能低め。

**黒 Black** — **最強**。ゲーム最速、ロケット2発、地雷2個。砲塔は強く狙い、**自機の移動先（予測位置）を狙う**。防御的で撃たれると逃げる。広い盤面ほど強い。

### 5.3 AI実装の共通要素

- 移動する敵は止まらず動き続け、不動の敵（茶・緑）を押して動かせる。
- 跳弾を撃つ敵（緑・紫・白・黒の一部）は、壁の反射を計算して射線を作る「跳弾計算ロジック」を共有。
- 「自機の現在位置を狙う」型（青緑など）と「移動先を予測して狙う」型（黒・緑）を分けて実装。
- AIは一定間隔で「移動方向の再決定」「射撃判断」「地雷判断」を行う（敵ごとに頻度・しきい値が異なる）。

## 6. ステージ／ミッション仕様

- **盤面データ**: グリッド（例: 20×16セル）でブロック・穴・破壊可ブロック・各戦車の初期配置・向きを定義。JSON的なデータ構造で持つ。
- **固定ミッション**: 1〜20、および21以降の10の倍数（30,40,…100）は手作りデータ。原作の代表的な配置を再現（例: Lv.50はLv.5の盤面流用＋黒戦車2台）。
- **ランダム生成**: 21以降の10の倍数以外は、過去の盤面を流用しつつ敵の種類・数・配置をランダム化するジェネレータを実装。出現する敵はミッション番号に応じて強い敵を含める。
- ミッション開始時に「MISSION XX」表示と残機表示。クリア／ゲームオーバー演出。

## 7. 画面・UI仕様

| 画面 | 内容 |
|---|---|
| タイトル画面 | モード選択（1人プレイ / 2人対戦）、操作説明、設定 |
| ゲーム画面 | フィールド描画、残機・ミッション番号・残り敵数・スコアのHUD |
| ポーズ画面 | 再開 / リスタート / タイトルへ戻る |
| ミッションクリア | 「CLEAR」表示 → 次ミッションへ |
| ゲームオーバー | 到達ミッション・スコア表示 → タイトルへ |
| リザルト（2人対戦） | 各プレイヤーの撃破数と勝敗表示 |

- 戦車・砲弾・地雷・壁はシンプルな図形（矩形・円）で描画。色は原作の各戦車色に準拠。
- 自機は鮮やかな単色、敵はくすんだ色＋履帯ストライプで区別（P2赤と敵ピンクの混同防止）。
- 効果音（任意・段階2）: 発射・爆発・クリア・白戦車の透明化音など。Web Audio APIで簡易生成または短い音源。

## 8. 技術アーキテクチャ

### 8.1 ファイル構成

```
index.html   … 単一ファイル（HTML + <style> + <script>）
```

> 開発中はモジュールを論理的に分けて記述し、最終的に1ファイルへ集約する。

### 8.2 推奨モジュール（論理構成）

| モジュール | 役割 |
|---|---|
| `GameLoop` | requestAnimationFrameによるメインループ（update→render） |
| `InputManager` | キーボード・マウス入力の状態管理 |
| `Tank` | 戦車の基底クラス（位置・向き・砲塔角・速度・HP） |
| `PlayerTank` / `EnemyTank` | 自機・敵の派生。敵は種別ごとのAIを持つ |
| `EnemyAI` | 種別別の移動・射撃・地雷ロジック（青緑/緑/黒等の関数群） |
| `Bullet` | 砲弾（直進・跳弾回数・寿命・衝突） |
| `Mine` | 地雷（タイマー・近接反応・爆発範囲） |
| `Field` / `Tile` | 盤面・ブロック・穴・破壊可ブロックの管理と当たり判定 |
| `Collision` | AABB判定・反射ベクトル計算・跳弾計算 |
| `MissionManager` | ミッションデータの読込・ランダム生成・進行管理 |
| `UI` / `Scene` | タイトル・ゲーム・ポーズ・リザルトの画面遷移 |
| `Audio` | 効果音（段階2） |

### 8.3 当たり判定・物理

- 戦車↔壁: AABB。めり込み補正で滑らかに移動。
- 砲弾↔壁: 接触面の法線で速度ベクトルを反射。跳弾カウントを減算、0で消滅。
- 砲弾↔戦車: 円or矩形判定で即破壊。
- 砲弾↔砲弾: 接触で相殺。
- 地雷: 近接（半径判定）・タイマー・被弾で爆発。爆発は範囲内の戦車・地雷・破壊可ブロックに作用。

## 9. 段階的開発ロードマップ

フル再現を以下の段階で積み上げる（各段階で動作確認可能な状態にする）。

### フェーズ1: コア基盤（MVP）
- Canvas描画・ゲームループ・入力処理
- 自機の移動・砲塔エイム・砲弾発射・跳弾
- 壁との当たり判定、敵=茶（不動）1種、1ステージクリア判定

### フェーズ2: 戦闘の骨格
- 地雷・破壊可ブロック・穴
- 灰・青緑・ピンクの追加（移動AIと射撃AI）
- 残機制・ミッション進行（固定の数ステージ）

### フェーズ3: 全敵・全ルール
- 黄・緑・紫・白・黒の追加（跳弾計算・予測射撃・透明化）
- ミッション1〜20の固定盤面、5ミッションごと残機+1
- 100ミッション化＋ランダム生成＋10の倍数固定盤面

### フェーズ4: 2人対戦・仕上げ
- ローカル2人対戦モード（操作・勝敗・リザルト）
- タイトル/ポーズ/ゲームオーバー画面、HUD整備
- 効果音、難易度・速度の調整

## 10. 受け入れ基準（完成の定義）

- [ ] ブラウザで `index.html` を開くだけで起動する。
- [ ] 1人プレイで自機をWASD移動・マウスエイム・クリック射撃でき、跳弾が機能する。
- [ ] 9種すべての敵が登場し、それぞれ表5.1・5.2どおりの挙動をする。
- [ ] 地雷・破壊可ブロック・穴が仕様どおり機能する。
- [ ] 残機制・5ミッションごと残機+1・100ミッション進行・ランダム生成が動作する。
- [ ] ローカル2人対戦が成立し、撃破数で勝敗が決まる。
- [ ] 被弾で即破壊、全敵撃破でクリア、残機0でゲームオーバーが正しく判定される。
- [ ] 60fps程度で安定動作する。

---

## 11. 非対象（やらないこと）

- オンライン／ネットワーク対戦
- Wiiリモコン等の専用デバイス対応
- アカウント・サーバー保存（スコアはセッション内のみ）
- Mii連携・原作のグラフィック／音源の流用（著作権配慮のため自前のシンプルな図形・音で表現）

---

## 12. Claude Codeへの実装指示メモ

- まず `index.html` の雛形（Canvas＋ゲームループ＋入力）を作り、フェーズ1から順に積み上げること。
- 敵AIは `EnemyAI` に種別ごとの関数として分離し、性能パラメータは表5.1をデータ化して参照する。
- 跳弾計算は共通関数化し、緑・黒など予測射撃する敵から再利用する。
- 各フェーズ完了時に、その時点で遊べる状態を保つ（壊れた中間状態を残さない）。
- マジックナンバー（速度・クールダウン・爆発半径等）は定数として上部にまとめ、調整しやすくする。
